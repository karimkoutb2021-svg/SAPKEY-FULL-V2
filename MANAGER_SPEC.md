## 1. نظرة عامة على النظام

### 1.1 الرؤية
منظومة المدير الذكي (SMD) هي لوحة تحكم متكاملة Apple-Quality لإدارة السوبر ماركت بالكامل، تجمع بين:
- التشغيل اليومي الميداني (POS، الخزائن، الجرد)
- إدارة التجارة الإلكترونية (طلبات الأونلاين، التوصيل)
- الرقابة المالية والمحاسبية (الخزائن، المحافظ، المصروفات)
- إدارة الموارد البشرية (الحضور، المهام، الأذونات)
- التكامل الصوتي والذكاء الاصطناعي (Voice/OCR/AI)

### 1.2 المبادئ الأساسية
1. **Mobile-First Responsive** — تعمل بكفاءة فائقة على الكمبيوتر والموبايل
2. **Offline-First** — جميع العمليات الحساسة تدعم العمل بدون إنترنت
3. **Real-Time** — تحديثات حية عبر Supabase Realtime Channels
4. **Bilingual** — واجهة عربية/إنجليزية كاملة مع RTL
5. **Voice-First** — التحكم الصوتي في جميع العمليات التشغيلية
6. **Role-Based** — صلاحيات مختلفة حسب الدور (كاشير، مدير، أدمن)
7. **RTL (Right-to-Left)** — إجبارية الاتجاه من اليمين لليسار

### 1.3 المستخدمون المستهدفون
- **مدير الفرع (Manager):** جميع الصلاحيات التشغيلية ما عدا اعتماد القيود المحاسبية
- **المدير المالي (Admin):** صلاحيات شاملة مع اعتماد التسويات المخزنية والمالية
- **الكاشير (Cashier):** صلاحيات البيع فقط مع أدوات محدودة (الكودينج، الجرد، الليبل)
- **مندوب التوصيل (Driver):** صلاحيات محدودة لتحديث حالة الطلبات

   - الأصناف منخفضة المخزون
   - رصيد الخزينة الحالي
   - الموظفين النشطين حالياً

2. **الإجراءات السريعة (4 أزرار):**
   - إيداع نقدي
   - سحب نقدي
   - جرد صوتي
   - جرد بالكاميرا

3. **النشاط الأخير:**
   - آخر 10 عمليات (حركات خزينة، طلبات، جرد)
   - عرض زمني محدث تلقائياً

4. **التكامل مع Toolbox:**
   - Ctrl+K → Command Palette
   - أدوات الكودينج، الجرد، الليبل، الباسپورت، الإعتماد

#### 5.2.2 مراقبة نقاط البيع (POS Monitoring) — جديد
**المسار:** `/manager/pos-monitoring`

**الوصف:**
شاشة حية تعرض الكاشيرات النشطة وفواتيرهم اللحظية في الوقت الفعلي.

**المكونات:**
1. **شريط إحصائيات الـ POS:**
   - إجمالي مبيعات اليوم
   - عدد الفواتير المكتملة
   - الكاشيرات النشطة
   - متوسط قيمة الفاتورة

2. **جدول الكاشيرات النشطين:**
   - اسم الكاشير
   - حالة الوردية (مفتوحة/مغلقة)
   - عهدة البداية
   - إجمالي المبيعات في الوردية
   - عدد الفواتير
   - القيمة النقدية + الشَّبكة (الفيزا)
   - زمن بدء الوردية

3. **آخر فواتير الـ POS:**
   - رقم الفاتورة
   - الكاشير
   - العميل
   - المبلغ
   - طريقة الدفع
   - الحالة
   - الوقت

4. **حركة النقدية اليومية:**
   - رسم بياني بسيط (إيداع/سحب/مبيعات)
   - إجمالي المقبوضات النقدية
   - إجمالي مدفوعات الشَّبكة

#### 5.2.3 الخزينة والمحافظ (Treasury) — محسّن
**المسار:** `/manager/treasury`

**التحسينات المطلوبة على القائم:**
1. **إضافة حساب جديد:** يمكن المدير من إنشاء خزائن فرعية تشغيلية أو شخصية
2. **رفع المرفقات:** إرفاق صور السكرين شوت للتحويلات المالية
3. **تصفية متقدمة:** تصفية الحركات حسب النوع والحالة والتاريخ
4. **القروض الداخلية (Loans):** إدارة السلف للموظفين
5. **عهدة المدير الذكية:** عداد ينخفض مع كل صرف نثرية

#### 5.2.4 إدارة الطلبات (Orders) — محسّن
**المسار:** `/manager/orders`

**التحسينات المطلوبة على القائم:**
1. **خط Pipeline متطور:** أعمدة Drag & Drop للحالات
2. **تعيين مندوب التوصيل:** اختيار المندوب من قائمة مباشرة
3. **إشعارات WhatsApp:** إرسال تحديث تلقائي عند تغيير الحالة
4. **تتبع وقت التحضير:** حساب زمن إعداد الطلب
5. **تقييمات العملاء:** عرض تقييمات الطلبات المكتملة
6. **إدارة المرتجعات:** قبول أو رفض المرتجعات مع تحديث المخزون

#### 5.2.5 المصروفات النثرية (Expenses) — جديد
**المسار:** `/manager/expenses`

**المكونات:**
1. **إضافة مصروف نثري:**
   - حقل المبلغ
   - حقل البيان/السبب
   - اختيار القسم المستفيد
   - اختيار الموظف المسؤول
   - كاميرا الهاتف لتصوير الإيصال
   
2. **عهدة المدير الذكية:**
   - عداد رصيد العهدة
   - تناقص تلقائي مع كل صرف
   - حد الأمان → تنبيه ذكي
   
3. **طلب تعويض العهدة:**
   - زر لتجميع النثريات المفتوحة
   - إرسال تقرير مؤتمت للإدارة المالية

4. **سجل المصروفات:**

## 7. معيار شجرة وحدات القياس

### 7.1 الوحدات الأربع الرئيسية

```
┌─────────────────────────────────────────────────────┐
│           معيار وحدات القياس الموحدة                   │
├─────────────────────────────────────────────────────┤
│                                                     │
│  1. الوحدة الأساسية (Base Unit / Each)              │
│     - أصغر وحدة قياس                                │
│     - قطعة، عبوة، كيس، علبة                         │
│     - يتم الجرد والبيع بها للمستهلك النهائي          │
│     - بناءً عليها تُحسب التكلفة وسعر البيع            │
│                                                     │
│  2. وحدة الوزن (Weight Unit)                        │
│     - كيلوجرام (Kg) / جرام (Gm)                     │
│     - للمنتجات التي تباع وتشترى بالميزان             │
│     - لحوم، أجبان، خضروات، فواكه                     │
│                                                     │
│  3. وحدة الحجم (Volume Unit)                        │
│     - لتر (Ltr) / ملي لتر (Ml)                       │
│     - للمنتجات السائلة                               │
│     - مياه غازية، عصائر، زيوت، منظفات                │
│                                                     │
│  4. وحدة التعبئة (Pack / Carton / Box)              │
│     - لتسهيل عمليات الشراء بالجملة                   │
│     - تتبع المخزون في المستودعات                     │
│     - معامل تحويل تلقائي للوحدة الأساسية             │
│     - مثلاً: كرتونة = 24 علبة                        │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 7.2 قواعد التكامل المحاسبي
1. سعر التكلفة دائماً مضبوط على الوحدة الأساسية
2. سعر البيع مضبوط على الوحدة الأساسية
3. عند البيع بالوزن، يتم وزن المنتج وتحويل القيمة
4. عند استلام مشتريات من مورد، يتم إدخال كرتونة وتحويلها تلقائياً

## 8. منظومة الأدوات الذكية (Toolbox)

### 8.1 Command Palette (لوحة الأوامر)
- اختصار: Ctrl+K / ⌘K
- بحث Spotlight-style مع تطبيع عربي
- نتائج حسب الدور (الكاشير يرى أدوات محدودة)
- 6 أوامر أساسية: Coding, Audit, Label, Passport, Approve, OCR
- تتبع الأوامر الأخيرة (آخر 5)
- تنقل بالأسهم واختيار بـ Enter

### 8.2 Coding Wizard (معالج التكويد الصوتي)
- 7 خطوات صوتية:
  1. كود المنتج
  2. اسم الصنف
  3. رقم الرف
  4. القسم
  5. الوحدة
  6. سعر التكلفة
  7. سعر البيع
- تحويل الأرقام المنطوقة (مصرية) إلى أرقام
- رسوم متحركة انزلاقية بين الخطوات
- مساران: مدير → مسودة مباشرة، كاشير → بانتظار المراجعة

### 8.3 Audit Panel (لوحة الجرد)
- 3 تبويبات: صوتي، يدوي، تصوير
- Product Passport (بطاقة المنتج)
- تحديث المخزون تلقائي (للمدير فقط)
- إظهار الفروق بالألوان (أخضر/أحمر/كهرماني)

### 8.4 Label Print (طباعة اللاصقات)
- بحث عن المنتج
- طباعة CODE128 باركود
- سعر المنتج
- مقاس 100mm × 60mm

## 9. حالات الاستخدام والتدفقات

### 9.1 تدفق الجرد الصوتي
```
1. يفتح المدير الجرد الصوتي من Toolbox (Ctrl+K)
2. يختار علامة التبويب "صوتي"
3. يتكلم باسم المنتج ← النظام يبحث بالصوت
4. يظهر المنتج مع الكمية المسجلة في النظام
5. يدخل الكمية الفعلية على الرف
6. النظام يحسب الفرق ويعرض النتيجة
7. المدير (فقط) يمكنه تحديث المخزون تلقائياً
8. يسجل الجلسة في audit_sessions
```

### 9.2 تدفق التكويد الصوتي
```
1. يفتح المدير الكودينج من Toolbox
2. المساعد الصوتي يطلب كود المنتج
3. المستخدم يتكلم ← النظام يحول الأرقام
4. ينتقل تلقائياً للخطوة التالية
5. بعد 7 خطوات، يعرض ملخص البيانات
6. المدير يؤكد ← تنشأ مسودة pending
7. الأدمن يعتمد المسودة ← يصبح المنتج نشطاً
```

### 9.3 تدفق إدارة المصروفات النثرية
```
1. المدير يفتح صفحة المصروفات
2. يضغط "إضافة مصروف"
3. يدخل المبلغ، البيان، القسم، الموظف
4. يصور الإيصال بالكاميرا
5. العهدة الذكية تتناقص تلقائياً
6. عند وصول العهدة لحد الأمان ← تنبيه
7. زر "طلب تعويض" يرسل تقرير للإدارة المالية
```

### 9.4 تدفق طلب الأونلاين
```
1. عميل يقدم طلباً عبر الموقع
2. يظهر الطلب في Pipeline (قسم الطلبات)
3. المدير يؤكد الطلب ← تحديث الحالة
4. ينطلق إشعار WhatsApp للعميل
5. الفريق يجهز الطلب ← تحديث إلى "جاهز"
6. يُعيَّن مندوب توصيل
7. المندوب يستلم ويوصل ← تحديث إلى "تم التسليم"
8. ينطلق إشعار نهائي للعميل
```
